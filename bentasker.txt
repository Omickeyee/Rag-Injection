                <div class="metadata">
                    <p class="byline author vcard"><span class="byline-name fn" itemprop="author">
                        Ben Tasker
                    </span></p>
            <p class="dateline">
            <a href="posts/blog/general/adding-human-json-to-my-site.html" rel="bookmark">

                </div>
            </header><div class="p-summary entry-summary">

                    <div>
<p>Recently, I've seen <a href="https://neilzone.co.uk/2026/03/implementing-the-somewhat-whimsical-human-json-protocol-on-my-website/">posts</a> by a couple of people about <a href="https://shkspr.mobi/blog/2026/03/adding-human-json-to-wordpress/">adding <code>human.json</code> to their sites</a>.</p>
<p>At it's core, <code>human.json</code> is a simple attestation that the content they publish is written by a human and not by AI.</p>
<p>Obviously, any AI generated site could also publish a <code>human.json</code>, so <a href="https://codeberg.org/robida/human.json">the system</a> works as a web of trust - the user trusts Alice, who states that Bob is a human, allowing the user to also trust Bob.</p>
<p>There are a couple of browser addons (<a href="https://addons.mozilla.org/en-US/firefox/addon/human-json-verifier/">for Firefox</a> and <a href="https://chromewebstore.google.com/detail/humanjson-verifier/haemonkhjllbfbilegdanbcdaednphko?authuser=0&amp;hl=en">for Chrome</a>) which can display an indicator of humanity.</p>
<p>This short post covers creating a script to easily add <code>human.json</code> generation to the build workflow of my sites.</p>

            <time class="published dt-published" datetime="2026-03-21T10:36:00Z" itemprop="datePublished" title="2026-03-21 10:36">2026-03-21 10:36</time></a>

                </div>
            </header><div class="p-summary entry-summary">
		        <img itemprop="image" src="images/BlogItems/booklore-to-cwa-migration/calibre-web.png" class="introimage" style="display: none"><div>
<p>A few weeks ago, I <a href="posts/blog/general/moving-ebooks-to-booklore.html">migrated our e-book library to Booklore</a>. Unfortunately, it seems that I backed the wrong horse: Booklore has since <a href="https://old.reddit.com/r/selfhosted/comments/1rvmpnv/booklore_is_gone/">ceased to exist</a>.</p>
<p>Unfortunately, toxicity <a href="posts/blog/the-internet/toxicity-in-software-related-discussions.html">isn't uncommon</a> in open source projects, so I'm not going to add to it by talking much about the context behind Booklore being deleted.</p>
<p>However, as it's relevant to a decision that I made in this process, I <em>will</em> say that I was already <a href="https://mastodon.bentasker.co.uk/@ben/116198743204185593">a little concerned</a> about the ongoing quality of Booklore, so it doesn't surprise me <em>too much</em> to hear that other people were being quite vociferous about the developer's use of AI coding tools (amongst other, IMO bigger, <a href="https://www.reddit.com/r/selfhosted/comments/1rs275q">issues</a>).</p>
<p>Whatever the rights or wrongs of it, Booklore's repo no longer exists, having been deleted along with the project's Discord and <a href="https://web.archive.org/web/20260312214732/https://booklore.org/">website</a>.</p>
<p>There <strong>is</strong> a fork (called <a href="https://github.com/grimmory-tools/grimmory">Grimmory</a>) that I could have switched to. However, given that I already had some quality concerns with Booklore, I decided that I didn't want to gamble on the new fork's ability to clear through tech debt. I'll <em>probably</em> check back in six months to see how things have progressed though.</p>
<p>Instead, I decided to move to <a href="https://github.com/crocodilestick/Calibre-Web-Automated">Calibre-Web Automated</a> (CWA). It's a fork of <a href="https://github.com/janeczku/calibre-web">Calibre-web</a> with some additional functionality like Smart shelves and KOReader sync support.</p>
<p>Both Calibre-Web and CWA consume an existing Calibre format library.</p>
<p>Although I still had my original Calibre library, I'd since added a number of new books to Booklore. Rather than risking mistakes by manually identifying and adding them, I decided to migrate the entire library from Booklore into Calibre-Web as if I'd simply had a filesystem directory full of ebooks.</p>

<p>Earlier this week, Cloudflare <a href="https://developers.cloudflare.com/changelog/post/2026-03-10-br-crawl-endpoint/">announced</a> the introduction of their Browser Crawl Endpoint.</p>
<p>This allows Cloudflare users to crawl an entire website by making a <em>single</em> API call to the Browser rendering service.</p>
<p>Although the browser rendering service <a href="https://developers.cloudflare.com/browser-rendering/reference/robots-txt/#best-practices-for-robotstxt">honours robots.txt</a> they don't define a specific User-Agent that the service will check for, apparently instead expecting website operators to disallow <strong>all</strong> user agents if they want to keep Cloudflare out.</p>
<p>However, they have also documented that the service includes Cloudflare specific request headers, allowing requests to be blocked by checking for those.</p>
<p>This post details how to achieve that on <a href="https://bunny.net/">BunnyCDN</a>, Nginx and Openresty.</p>

            </header><div class="p-summary entry-summary">
		        <img itemprop="image" src="images/Documentation/kobo_koreader/ko-reader.jpg" class="introimage" style="display: none"><div>
<p>This week, I <a href="posts/blog/general/moving-ebooks-to-booklore.html">set up a Booklore instance</a> to allow me to wirelessly sync ebooks to my Kobo e-reader.</p>
<p>However, in the (short) time since, I've run into a bit of a limitation: Booklore's Kobo sync <a href="https://github.com/booklore-app/booklore/issues/2533">doesn't support PDF files</a> (which seems to be a limitation of the way that Kobo fetches files from its store rather than Booklore specific).</p>
<p>Most of the <em>books</em> that I read are in EPUB format, but I've also got various user manuals and technical reference docs in PDF that I'd like to be able to fetch onto the reader, on demand.</p>
<p>Although I <em>could</em> use a USB cable to sync PDFs over, it'd somewhat undermine the point in running Booklore in the first place.</p>
<p>So, I decided to install <a href="https://koreader.rocks/">KOReader</a> onto my Kobo. </p>
<p>KOReader is a document reader designed for e-ink devices and can fetch books (and more importantly, PDFs) from Booklore via OPDS, syncing reading progress back via Booklore's dedicated KOReader api.</p>
<p>This documentation details how to install KOReader onto a Kobo before configuring it to work with Booklore. If you're a Calibre user, KOReader can also <a href="https://dmpop.github.io/koreader-compendium/13-calibre-connect/">talk to Calibre</a>.</p>

<p>I own an <a href="https://sportsmk.co.uk/cat/66/air-rifles/smk-air-rifles/#38947">SMK CP2 Convertible CO<sub>2</sub> gun</a>. Also sold under other names (including Artemis CP2 and Diana Chaser) the gun's unique selling point is that it ships with two barrels allowing easy conversion between rifle and pistol forms.</p>
<p>As you might expect, it <em>can</em> be quite a satisfying gun to shoot. Unfortunately, though, the use of CO<sub>2</sub> as a propellant does come with drawbacks. </p>
<p>The pressure that drives the gun is derived from liquid CO<sub>2</sub> in the cartridge transitioning to gas. The rate at which this conversion occurs is influenced by temperature: the lower the temperature the slower the output pressure recovers.</p>
<p>It's not just ambient temperature that's a factor: CO<sub>2</sub> is a refrigerant, so as it expands, the temperature of the cartridge drops, with a knock on impact on pressure. Although the cartridge eventually warms back up, this dynamic limits the rate at which the gun can effectively be fired.</p>
<p>Temperature variances affect the gun's output power which, in turn, affects <em>accuracy</em> across shots.</p>
<p>Putting all of this together meant that, sat outside in the British winter, I had some <em>quite frustrating</em> sessions with the gun - to the extent that I quite strongly considered selling it on.</p>
<p>Instead, though, I decided to try something else.</p>
<p>This post talks about my experiences converting an SMK CP2 to use regulated high pressure air (HPA).</p>

<p>Although I use this for airguns, the chronograph apparently works for rim-fire and even archery too: this should therefore also work for those sports.</p>
